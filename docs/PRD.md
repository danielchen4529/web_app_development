# 食譜收藏夾系統 PRD (Product Requirements Document)

## 1. 專案概述
- **背景與動機**：在網際網路上經常看到各式各樣吸引人的食譜，但分散在不同的網站、部落格或社群媒體中。為了方便日後查詢與製作，需要一個集中管理的個人食譜收藏夾，統一收納所有網路上看到的食譜。
- **目標用戶**：自己（料理愛好者、需要個人食譜管理系統的使用者）。
- **核心價值主張**：提供一個輕量、直覺的個人化食譜儲存庫，支援圖片匯入與關鍵字搜尋，讓找食譜、看食譜、做料理的體驗更流暢。

## 2. 功能需求

以下列出系統的主要功能與對應的使用者故事（User Stories）：

1. **新增與編輯食譜**
   - 作為使用者，我希望「能夠手動新增食譜（包含標題、食材清單、步驟說明等）」，以便「將網路上的食譜內容整理成自己習慣的格式」。
   - 作為使用者，我希望「能隨時編輯已儲存的食譜」，以便「根據個人的料理經驗隨時調整調味料比例或料理步驟」。

2. **圖片匯入與展示**
   - 作為使用者，我希望「能上傳或透過網址匯入食譜成品的圖片」，以便「在瀏覽清單時可以快速被視覺吸引，回想起那道料理」。

3. **關鍵字搜尋**
   - 作為使用者，我希望「能透過關鍵字搜尋食譜標題與食材」，以便「當我想消耗冰箱裡的特定食材時，能快速找到對應的食譜」。

4. **食譜分類與標籤（Tagging）**
   - 作為使用者，我希望「能為食譜加上分類（如：中式、甜點、晚餐）或標籤」，以便「未來能更有系統地瀏覽和管理食譜庫」。

5. **食譜瀏覽與閱讀模式**
   - 作為使用者，我希望「在檢視單一食譜時能有清晰、易讀的排版」，以便「在廚房實際做菜時，可以方便地用手機或平板看著步驟操作」。

## 3. 非功能需求
- **技術限制**：
  - 後端：Python Web 框架使用 **Flask**。
  - 前端模板：使用 **Jinja2** 進行頁面渲染，搭配純 CSS 或輕量化 CSS 框架。
  - 資料庫：使用 **SQLite** 作為關聯式資料庫，方便本機開發與輕量部署。
- **效能與安全考量**：
  - **效能**：搜尋回應時間應在合理範圍內；圖片若有上傳需求，應考慮檔案大小限制以節省空間與提升載入速度。
  - **安全**：防範常見的 SQL Injection 及 XSS 攻擊（透過 Jinja2 預設的脫逸機制）。由於系統初步設定為個人使用，可以先採單機或簡易登入防護，未來若公開佈署需考量完善的權限驗證。

## 4. MVP 範圍 (Minimum Viable Product)

| 層級 | 功能描述 |
| --- | --- |
| **Must Have (必須有)** | - 建立食譜的基本表單（標題、食材、步驟的純文字輸入）。<br>- 食譜列表檢視與詳細頁面檢視。<br>- 單一圖片上傳或網址連結功能。<br>- 基本關鍵字搜尋。 |
| **Should Have (應該有)** | - 食譜編輯與刪除功能。<br>- 基本的食譜分類。 |
| **Nice to Have (最好有)** | - 根據外部網址自動爬取食譜內容（Auto-fetch）。<br>- 用餐人數份量換算工具。<br>- 響應式介面 (RWD) 最佳化（手機觀看更好用）。 |
